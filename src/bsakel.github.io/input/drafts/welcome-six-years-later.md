# Six Years Later, I Finally Pressed Publish

![Commit history that restarted this project](/assets/images/blog-repo-commit-history.png)

That screenshot is where this story really begins.

Welcome to my blog.

This space is for sharing anything that seems interesting to me: engineering lessons, useful tools, team and delivery insights, and occasionally the messy stories behind shipping real work.

For a first post, I wanted to start with one of those messy stories.

This blog idea is six years old.

This blog post is new.

## The Spark

I found `terminal.css` and had that immediate builder thought:

“This is it. I want my blog to look like this.”

Minimal. Focused. No noise. Just writing and ideas.

So I did what most of us do when momentum appears: I went to create a GitHub repo.

Except I already had one.

## The Surprise in My Own Repository

As soon as I opened it, I remembered.

I had started this exact project years ago.

Back then, I had done the usual optimistic setup: choose a stack, wire the basics, plan to write consistently, and tell myself “I’ll finish this soon.”

Then life happened the way it always does: shifting priorities, tighter schedules, responsibilities that matter more in the moment.

The project didn’t fail.

It just went quiet.

## The 3-4 Hour Restart

Instead of overthinking it, I treated the restart as a focused session.

I gave AI two things:

- the CSS style I wanted
- the existing repository

Then we worked through it together for about 3 to 4 hours.

Not in a “push one button and walk away” way.

More like fast pair-programming: making decisions, tightening structure, simplifying pages, and shaping the blog into something I actually wanted to publish.

For the first time in a long while, this project felt close.

## The 80% Trap

If you build software, you know this moment.

Everything looks almost done. You can see the finish line.

And that’s exactly when hidden constraints show up.

In my case, the old setup was based on `Statiq.Web`, and I rediscovered a limit of 10 posts.

To be fair, `Statiq.Web` is a paid tool, and that limit was part of that reality. I had simply forgotten about it after all this time.

But in practice, it stopped me right before launch.

The timing couldn’t have been worse.

## The 10-Minute Pivot

This was the turning point.

Instead of getting stuck (again), AI helped me migrate the project from `Statiq.Web` to `Statiq.Framework` in roughly 10 minutes.

That quick migration removed the blocker and preserved momentum.

And momentum is everything for long-paused projects.

The task itself wasn’t glamorous, but it was exactly the kind of work that can quietly kill a shipping day if it drags on.

This time, it didn’t.

## Why This Is My First Post

Because this is what I want the blog to represent.

Not perfection.

Progress.

The honest version of building: delayed starts, forgotten constraints, quick recoveries, and finally publishing anyway.
